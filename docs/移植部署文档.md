# 网络安全资产扫描系统 - 移植部署文档

## 一、服务器配置要求

### 1.1 硬件要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核+ |
| 内存 | 2GB | 4GB+ |
| 磁盘 | 10GB | 50GB+（长期存储扫描数据） |
| 网络 | 100Mbps | 1Gbps（扫描效率与带宽相关） |

### 1.2 操作系统要求

| 操作系统 | 版本要求 | 说明 |
|---------|---------|------|
| **Windows** | Windows Server 2016+ / Windows 10+ | 开发环境，直接运行 |
| **Linux（推荐）** | CentOS 7+ / Ubuntu 18.04+ / Debian 10+ | 生产环境推荐 |
| **国产信创** | 银河麒麟V10 / 统信UOS / openEuler | 兼容，需验证Python版本 |

> ⚠️ 系统使用 TCP Socket 进行端口扫描，需要有足够的文件描述符（Linux 默认1024可能不够）

### 1.3 Linux 系统调优

```bash
# 临时调整文件描述符限制
ulimit -n 65535

# 永久调整
echo "* soft nofile 65535" >> /etc/security/limits.conf
echo "* hard nofile 65535" >> /etc/security/limits.conf

# 调整内核参数（高并发扫描优化）
echo "net.ipv4.tcp_tw_reuse = 1" >> /etc/sysctl.conf
echo "net.ipv4.ip_local_port_range = 1024 65535" >> /etc/sysctl.conf
sysctl -p
```

## 四、网络与端口要求

### 4.1 服务端口

| 端口 | 协议 | 用途 | 配置项 |
|------|------|------|--------|
| 20260 | HTTPS | Web 管理界面 | `config.py → WEB_PORT` |

### 4.2 扫描端口（出站）

系统需要对目标IP发起 TCP 连接探测，需要服务器到目标网络的出站访问权限。

常见扫描端口范围：
- 快速模式：115个常用端口（21-27017 + 49152-49215）
- 中速模式：1000+端口
- 全端口模式：1-65535

### 4.3 防火墙配置

```bash
# Linux firewalld
firewall-cmd --permanent --add-port=20260/tcp
firewall-cmd --reload

# Linux iptables
iptables -A INPUT -p tcp --dport 20260 -j ACCEPT

# Windows
# 在 Windows 防火墙中添加入站规则，允许 TCP 20260
```

## 五、部署步骤

### 5.1 方式一：直接运行（推荐测试环境）

```bash
# 1. 上传项目
scp -r NetScan/ user@server:/opt/netscan

# 2. 进入目录
cd /opt/netscan

# 3. 创建虚拟环境（推荐）
python3 -m venv venv
source venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt

# 5. 生成SSL证书（首次）
python gen_cert.py

# 6. 启动服务
python app.py
```

### 5.2 方式二：Systemd 服务（推荐生产环境）

```bash
# 创建服务文件
cat > /etc/systemd/system/netscan.service << 'EOF'
[Unit]
Description=网络安全资产扫描系统
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/netscan
ExecStart=/opt/netscan/venv/bin/python app.py
Restart=always
RestartSec=5
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
EOF

# 启用并启动
systemctl daemon-reload
systemctl enable netscan
systemctl start netscan
systemctl status netscan
```

### 5.3 方式三：Docker 部署

```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .
EXPOSE 20260

CMD ["python", "app.py"]
```

```bash
# 构建镜像
docker build -t netscan:latest .

# 运行容器
docker run -d \
  --name netscan \
  -p 20260:20260 \
  -v /opt/netscan/data:/app/data \
  -v /opt/netscan/certs:/app/certs \
  --restart=always \
  netscan:latest
```

## 六、配置文件说明

编辑 `config.py` 进行自定义配置：

```python
# Web服务
WEB_HOST = "0.0.0.0"          # 监听地址
WEB_PORT = 20260              # 监听端口
SECRET_KEY = "修改为随机密钥"   # 会话密钥（务必修改！）
USE_HTTPS = True              # 是否启用HTTPS

# 扫描参数
SCAN_TIMEOUT = 1.5            # 单端口超时（秒）
SCAN_THREADS = 200            # 并发扫描线程数
FULL_SCAN_THREADS = 500       # 全端口模式线程数
MAX_TARGETS = 5000            # 最大目标数
BATCH_SIZE = 50               # 批次大小

# 会话
SESSION_TIMEOUT = 1800        # 登录超时（秒）
```

## 七、数据备份与迁移

### 7.1 备份

```bash
# 数据库（核心数据）
cp data/netscan.db backup/netscan_$(date +%Y%m%d).db

# 漏洞规则库
cp vuln_db/vuln_rules.json backup/

# SSL证书
cp certs/ backup/certs/ -r
```

### 7.2 迁移到新服务器

```bash
# 1. 在旧服务器打包
tar czf netscan_backup.tar.gz data/ vuln_db/ certs/

# 2. 传输到新服务器
scp netscan_backup.tar.gz user@newserver:/opt/

# 3. 在新服务器解压覆盖
cd /opt/netscan
tar xzf /opt/netscan_backup.tar.gz
```

## 八、安全建议

| 项目 | 建议 |
|------|------|
| 默认密码 | 首次登录后立即修改 admin 密码 |
| SECRET_KEY | 修改为随机生成的长字符串 |
| HTTPS | 保持启用，使用正式证书替换自签名证书 |
| 访问控制 | 通过防火墙限制管理端口的访问源IP |
| 运行权限 | 建议以非 root 用户运行（端口>1024无需root） |
| 数据备份 | 定期备份 `data/netscan.db` |

## 九、故障排查

| 问题 | 排查方法 |
|------|---------|
| 服务启动失败 | 检查端口是否被占用：`netstat -tlnp \| grep 20260` |
| 扫描无结果 | 检查防火墙是否允许出站TCP连接 |
| 扫描速度慢 | 减少目标数量，或使用快速扫描模式 |
| 数据库锁死 | SQLite 并发写入有限，避免同时执行多个扫描任务 |
| 证书错误 | 重新生成证书：`python gen_cert.py` |
